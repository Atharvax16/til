# Quantum Chemistry — GNNs for Molecular Property Prediction

Notes on three foundational papers for learning molecular properties (energies, forces)
directly from atoms + 3D coordinates, instead of slow DFT or hand-engineered features.

Read them in this order — each one fixes a concrete limitation of the previous:

```
MPNN (2017)  →  SchNet (2017)  →  DimeNet (2020)
unify the      make it smooth     add angles /
framework      & physical         directionality
```

---

## 1. Neural MPNN — *Neural Message Passing for Quantum Chemistry*
Gilmer, Schoenholz, Riley, Vinyals, Dahl (Google Brain), ICML 2017 · arXiv:1704.01212

### Core idea
Reformulates ~8 existing graph models into **one common framework: the Message Passing
Neural Network (MPNN)**. A molecule is a graph (atoms = nodes, bonds = edges), and the
forward pass has two phases:

1. **Message passing** (T steps): each node's hidden state `h_v` is updated from messages
   gathered over its neighbors — `m_v = Σ M_t(h_v, h_w, e_vw)`, then `h_v = U_t(h_v, m_v)`.
2. **Readout**: a permutation-invariant function `R({h_v})` pools node states into one
   molecule-level prediction.

`M_t`, `U_t`, `R` are all learned. Invariance to graph isomorphism comes from the
permutation-invariant readout.

### Key results
- New **state of the art on QM9** (130k molecules, 13 quantum properties).
- Reaches **chemical accuracy on 11 / 13 targets**.
- ~10⁵× faster at inference than the DFT calculation it approximates.

### What it contributes / upgrades over prior work
- Unifies prior graph models (Duvenaud fingerprints, GG-NN, Interaction Networks, DTNN,
  graph convolutions, Laplacian/spectral methods) under a single abstraction → easier to
  compare and to invent new variants.
- Best variant uses a **set2set readout** and **edge networks** that condition messages on
  bond/distance features.
- Introduces a trick to use **larger node representations without extra compute/memory**.

### One thing to remember
The "message passing + readout" template that almost every later molecular GNN (including
the next two papers) is a special case of.

## 2. SchNet — *A continuous-filter CNN for modeling quantum interactions*
Schütt, Kindermans, Sauceda, Chmiela, Tkatchenko, Müller (TU Berlin), NeurIPS 2017 · arXiv:1706.08566

### Problem it fixes in MPNN
MPNN-style models use **discrete** inputs (e.g. one-hot bond types). That makes the
predicted potential energy surface (PES) **discontinuous** as atoms move — so you can't
use them for molecular dynamics (computing forces by differentiating energy).

### Core idea
Replace discrete graph convolutions with **continuous-filter convolutions (cfconv)**: the
filter is a *function of the interatomic distance* `W(r_i − r_j)`, generated by a small NN,
so atoms at arbitrary positions are handled smoothly (no grid, no one-hot bonds).

Architecture: atom-type embeddings → 3 **interaction blocks** (cfconv + atom-wise layers
with ResNet-style residuals, shifted-softplus nonlinearity) → atom-wise energy → pooling.

### What it upgrades over MPNN
- **Smooth, continuous PES** → energy is differentiable, so it predicts **forces** as
  `F_i = −∂E/∂r_i` and the force field is guaranteed **energy-conserving (curl-free)**.
- Respects more physics by construction: rotation/translation/permutation invariance for
  energy, **rotational equivariance** for forces.
- Works on **non-equilibrium** geometries → usable for **molecular dynamics (MD-17)**,
  which MPNN/enn-s2s could not do.
- No hand-crafted bond features — only nuclear charges + positions.
- Introduces the **ISO17** benchmark (chemical *and* conformational variation); shows
  training on forces improves generalization.

### One thing to remember
Continuous-filter convolutions: let a CNN-like model act on atoms at arbitrary 3D
positions and produce a smooth, energy-conserving force field.

## 3. DimeNet — *Directional Message Passing for Molecular Graphs*
Gasteiger (Klicpera), Groß, Günnemann (TU Munich), ICLR 2020 · arXiv:2003.03123

### Problem it fixes in SchNet/MPNN
Those models pass messages using **only pairwise distances** — they ignore **direction /
angles**. But classical empirical potentials need angular terms
(`E = E_bonds + E_angle + E_torsion + E_nonbonded`). Distance-only GNNs literally **cannot
distinguish** some different molecules with identical neighbor distances (e.g. one hexagon
vs. two triangles within a cutoff).

### Core idea
**Directional message passing**: embed the **messages** (directed atom pairs `m_ji`),
not the atoms. Each message carries a direction, so it's **rotationally equivariant**.
Messages are updated using the **angle** between incoming messages (analogous to belief
propagation): `m_ji` is updated from neighbors `m_kj` transformed by the angle ∠(k,j,i)
and distance `d_kj`.

Distances and angles are encoded with a **physically principled orthogonal basis** —
**spherical Bessel functions + spherical harmonics** — instead of Gaussian radial bases.

### What it upgrades over SchNet
- Adds **angular / directional information** → can directly learn the `E_angle` term and
  distinguish molecules a distance-only GNN provably cannot (links to higher-order
  Weisfeiler-Lehman expressiveness).
- **Spherical Bessel basis** beats Gaussian RBFs while using **<1/4 the parameters** (4×+
  smaller radial basis).
- **Twice continuously differentiable** (smooth, no ReLU) → suitable for MD forces, like
  SchNet but more accurate.

### Key results
- Beats previous GNNs on average by **76% on MD17** and **31% on QM9**.

### One thing to remember
Embed *directed messages* and let them interact through the *angle between them* — that's
what lets the model capture bond angles and beat distance-only GNNs.

## The big picture — what each step "upgraded"

| Paper | Year | Geometry used | Key new capability |
|-------|------|---------------|--------------------|
| **MPNN**   | 2017 | distances (+ discrete bonds) | unified message-passing framework; SOTA on QM9 |
| **SchNet** | 2017 | continuous distances | smooth PES → forces, energy conservation, MD |
| **DimeNet**| 2020 | distances **+ angles** | directional info, spherical basis, big accuracy jump |

Recurring themes: bake in **physical symmetries** (invariance/equivariance), avoid
hand-engineered features, and keep the energy **smooth & differentiable** so forces are
usable for molecular dynamics.
